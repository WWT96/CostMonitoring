# 异常成本监控系统云端部署运维说明

## 1. 项目技术栈

本系统是一个 Python 本地 Web 应用，主框架是 Streamlit，不是传统前后端分离项目。

核心技术栈如下：

| 类型 | 技术 |
| --- | --- |
| 开发语言 | Python |
| Web 框架 | Streamlit |
| 数据处理 | pandas、numpy |
| 异常检测 / 统计计算 | scikit-learn |
| 图表展示 | plotly |
| 本地数据库 | SQLite |
| Excel 读写 | openpyxl、xlsxwriter |
| HTTP / LLM 调用 | requests |
| 配置文件 | .env、settings.json |
| 日志目录 | logs/ |
| 默认端口 | 8501 |

本地运行时，系统会启动一个 Streamlit 服务，浏览器访问：

```text
http://127.0.0.1:8501
```

或：

```text
http://localhost:8501
```

## 5. 当前本地化封装方案

当前本地化封装方案固定为：

```text
Bat To Exe 启动器 + wheels 离线依赖 + 首次运行自动创建 venv
```

目标电脑只需要提前安装 Python。用户解压后双击启动器即可运行。

启动器会自动完成以下动作：

1. 检查项目目录和入口文件。
2. 检查本地 venv 是否存在。
3. 如果 venv 不存在，自动创建虚拟环境。
4. 优先从本地 wheels/ 目录离线安装依赖。
5. 启动 Streamlit 服务。
6. 自动打开浏览器访问 8501 端口。

这套方式主要服务于内网、本地电脑、无公网依赖的部署场景。

## 6. 云端部署建议

云端服务器部署时，不需要 Bat To Exe 启动器。运维可以把它当作一个普通 Python Web 服务部署。

### Windows Server 示例

```cmd
cd 项目目录
python -m venv venv
venv\Scripts\python -m pip install -r requirements.txt
venv\Scripts\python -m streamlit run app.py --server.address=0.0.0.0 --server.port=8501
```

### Linux Server 示例

```bash
cd 项目目录
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
streamlit run app.py --server.address=0.0.0.0 --server.port=8501
```

启动后，服务监听：

```text
0.0.0.0:8501
```

## 7. 生产环境建议

生产环境建议不要直接暴露 Streamlit 端口到公网，而是在前面增加：

```text
Nginx / 反向代理 / HTTPS / 访问控制
```

推荐结构：

```text
用户浏览器
    ↓
HTTPS 域名
    ↓
Nginx / 网关
    ↓
Streamlit 服务 8501
    ↓
SQLite / 本地文件 / LLM API
```

如果后续并发用户较多，或数据量继续扩大，可以进一步改造为：

```text
后台任务队列 + 预计算结果表 + 更稳定的数据库服务
```

例如将 SQLite 升级为 PostgreSQL / MySQL，并把耗时测算任务放入后台异步任务中。

## 8. 给运维的一句话说明

这是一个 Python + Streamlit 的单体数据分析应用，使用 SQLite 本地数据库和本地文件路径配置，依赖 pandas、scikit-learn、plotly 等数据分析组件。本地通过 venv 启动，默认监听 8501 端口；云端部署时可作为一个 Python Web 服务运行，再由 Nginx 反向代理到公网域名。
